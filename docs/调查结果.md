# PicoVR 2.0.13 兼容性补丁 - 调查结果

## 概述

PicoVR（`com.picovr.wing`）版本 2.0.13（构建 132，日期 2016-08-25）是早期 Pico 手机 VR 头显（Pico 1 / 1S、Pico U / U Lite）使用的旧版多功能应用。原始 APK 的 `targetSdkVersion=19`（Android 4.4 KitKat），在现代 Android 上无法安装或运行（测试设备：三星 SM-S918U，Android 15，SDK 35）。

本文档记录了发现的所有问题以及为让应用在现代设备上启动运行而应用的补丁。

## 发现的问题与应用的补丁

### 1. `INSTALL_FAILED_DEPRECATED_SDK_VERSION`（安装被阻止）

**原因：** Android 14+ 拒绝安装 `targetSdkVersion < 24` 的应用。

**补丁：** 在 `apktool.yml` 中将 `targetSdkVersion` 从 19 提升到 24，并在 `AndroidManifest.xml` 中更新 `platformBuildVersionCode`/`platformBuildVersionName`。

### 2. `sharedUserId="com.android.packageinstaller"`（安装/安全被阻止）

**原因：** `sharedUserId` 自 Android 10（API 29）起已弃用，在现代 Android 上会被拒绝。与系统包安装器共享 UID 也是安全方面的危险信号。

**补丁：** 从 `<manifest>` 标签中移除 `android:sharedUserId` 属性。

### 3. 带有 intent-filter 的组件缺少 `android:exported`（API 31+ 安装被阻止）

**原因：** Android 12+ 要求每个带有 `<intent-filter>` 的 `<activity>`、`<receiver>`、`<service>` 和 `<provider>` 都必须显式声明 `android:exported`。原始清单中有 18 个组件缺少该属性。

**补丁：** 为所有 18 个有 intent-filter 但没有 exported 属性的组件（启动器 Activity、推送接收器、Unity Activity、PicoPlayer 等）添加 `android:exported="true"`。

### 4. 重复的 `<uses-permission>` 条目（lint 警告）

**原因：** 多个权限（`ACCESS_NETWORK_STATE`、`GET_TASKS`、`SYSTEM_ALERT_WINDOW`）被声明了两次。

**补丁：** 对权限列表去重。

### 5. `MODE_WORLD_READABLE` SecurityException 崩溃（运行时）

**原因：** `WingApp.onCreate`（以及其他 10 个类）调用了 `getSharedPreferences(name, MODE_WORLD_READABLE)`（模式 `0x1`）或 `MODE_WORLD_WRITEABLE`（`0x2`）/ `MODE_MULTI_PROCESS`（`0x4`）。自 Android 7（API 24）起，`MODE_WORLD_READABLE` 会抛出 `SecurityException` 并导致应用启动时崩溃。

**补丁：** 编写了 Python 脚本（`patch_shared_prefs.py`），遍历每个 `.smali` 文件，查找 `getSharedPreferences(Ljava/lang/String;I)` 调用，向后追踪模式寄存器到其 `const/4` 赋值，并将任何非零模式重写为 `0x0`（`MODE_PRIVATE`）。修补了 11 个 smali 文件：

- `com/picovr/unitylib/UnityActivity.smali`
- `com/picovr/wing/BaseActivity.smali`
- `com/picovr/wing/WingApp.smali`
- `com/picovr/wing/lark/LarkManager.smali`
- `com/picovr/wing/psetting/PSettingCITActivity.smali`
- `com/picovr/wing/psetting/PSettingDisplayActivity.smali`
- `com/picovr/wing/psetting/PSettingDisplayActivity$5.smali`
- `com/picovr/wing/psetting/PSettingSelectMachine.smali`
- `com/picovr/wing/startup/CalibrationActivity.smali`
- `com/picovr/wing/utils/SPUtils.smali`
- `com/picovr/wing/utils/Utils.smali`

### 6. 距离传感器监听器中的 `ArrayIndexOutOfBoundsException`（运行时崩溃，启动后约 20 秒）

**原因：** `BaseActivity$MySensorEventListener.onSensorChanged` 无条件读取 `event.values[1]` 和 `event.values[2]` 用于调试日志。在现代设备上，距离传感器（`TYPE_PROXIMITY=8`）只返回单元素 `values` 数组，因此 `values[1]` 抛出 `ArrayIndexOutOfBoundsException: length=1; index=1` 并导致应用崩溃。

实际的距离逻辑只使用 `values[0]`；`[1]`/`[2]` 读取纯粹用于 `Log.d` 调试字符串。

**补丁：** 将 `aget v3, v0, v4`（values[1]）和 `aget v3, v0, v3`（values[2]）读取替换为 `const/4 v3, 0x0`，使调试日志打印 `0.0` 而不是崩溃。`:cond_1` 处的实际传感器逻辑（只触碰 `values[0]`）未更改。

### 7. 应用级加固标志（预防性）

添加到 `<application>`：

- `android:requestLegacyExternalStorage="true"` - 保留视频播放器/下载管理器基于 `File` 的旧式存储访问。
- `android:usesCleartextTraffic="true"` - 应用与 `http://video.picovr.com`（无 HTTPS）通信；没有此标志，API 调用在 API 28+ 上会失败。
- `android:extractNativeLibs="true"` - 捆绑的 `.so` 文件在旧工具链中以未压缩方式存储在 APK 中；强制提取可避免在不支持 APK 内原生库的设备上出现 `dlopen` 问题。

### 8. 原生库文本重定位（警告，非致命）

`libstreamingsdk_jni-armandroid-r4-gcc44-mt-1.1.0.so` 存在文本重定位，在 API 23+ 上被阻止。链接器记录错误但应用继续运行，因为流媒体 SDK 仅用于视频商店，不用于应用启动。这是已知限制 - 修复需要从源代码重新编译原生库，而我们没有源代码。

### 9. 缺少 `libpenguin.so`（警告，非致命）

日志显示 `Unable to open libpenguin.so: dlopen failed: library "libpenguin.so" not found.` 这是 Pico 设备上预期的 Pico 专用系统库。应用优雅降级，在没有它的情况下继续运行。

### 10. 后端服务器状态

`http://video.picovr.com/api3/` 返回 HTTP 500，响应体为空。视频商店/电影目录功能将无法使用，因为后端实际上已死亡。登录、本地视频播放和启动器 UI 仍然可以运行。

## 测试结果

测试设备：三星 SM-S918U（Galaxy S23 Ultra），Android 15，SDK 35，arm64-v8a（通过 32 位兼容运行 armeabi-v7a 库）。

| 步骤 | 结果 |
|---|---|
| 安装 | 成功 |
| 授予权限 | 成功 |
| 启动 `StartupAnimActivity` | 成功，冷启动约 734ms |
| 切换到 `PicoVRMainActivity` | 成功 |
| 运行 30 秒以上无崩溃 | 成功（无 `AndroidRuntime:E` 条目） |
| 后端 API（`video.picovr.com/api3`） | HTTP 500 - 服务器端已失效 |

应用启动、显示主 UI，不再崩溃。在线功能（电影商店、应用商店、登录）依赖于已失效的 `video.picovr.com` 后端，没有替代服务器将无法使用。本地视频播放和 2D 启动器应该仍然可用。

完整 VR 模式测试仍需要 Pico 1S 和合适的智能手机（距离传感器触发 2D<->VR 切换，通过 Lark 服务进行蓝牙控制器配对）。
